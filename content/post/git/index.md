Если вы когда-нибудь создавали копии папок с названиями `project_final`, `project_final_2`, `project_REALLY_final`, то вам срочно нужен **Git**. 

Система управления версиями позволяет не просто сохранять код, а делать «снимки» проекта в разные моменты времени. Если что-то сломалось (а это случается часто), вы всегда можете вернуться назад.


### Мой личный Топ-команд на каждый день:

1.  **`git status`** — ваша главная команда. Показывает, что изменилось и что «видит» Git.
2.  **`git add .`** — подготовить все изменения к сохранению.
3.  **`git commit -m "описание"`** — зафиксировать изменения. Старайтесь писать понятные описания, а не просто "fix".
4.  **`git push`** — отправить свои труды в облако (на GitHub).

### Трудности, с которыми я столкнулась:

Работа с Git — это не только `push` и `pull`. Иногда возникают конфликты веток или ошибки с путями (как те же `detached HEAD` или проблемы с подмодулями). Главное помнить: **в Git почти всё можно восстановить**, если не паниковать.

> **Совет:** Если ваш сайт на Hugo выглядит «криво», проверьте, запушили ли вы папку `public` и не забыли ли про файл `.nojekyll`.

Git — это дисциплина. Сначала кажется сложным, но через неделю вы уже не понимаете, как жили без него.
